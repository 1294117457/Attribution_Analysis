### 核心原则

1. **砍掉前端规划**——当前用 Streamlit 或纯 CLI 输出即可，等核心链路跑通再说
2. **砍掉 Docker**——本地开发阶段直接跑 Python，部署时再容器化
3. **保持 `requirements.txt`**——`pyproject.toml` 适合需要发布的包，当前用 `requirements.txt` 更简单
4. **不需要 repository 封装层**——直接用 SQL 查询已经足够简单，封装反而增加复杂度
5. **先跑通最小闭环**——一个脚本 `run_analysis.py` 就能触发整个链路

## 最小闭环脚本（核心入口）

这个文件是整个项目的"门面"，一行命令跑通整个链路：

```python
# backend/scripts/run_analysis.py
"""
一行命令触发完整分析链路：
采集数据 → 异常检测 → 情感分析 → 归因分析 → 输出报告
"""

import sys
from pathlib import Path

sys.path.insert(0, str(Path(__file__).parent.parent))

from app.data.akshare_client import fetch_klines
from app.analysis.anomaly_detector import detect_anomalies
from app.analysis.sentiment import analyze_sentiment
from app.analysis.attribution import generate_report


def run_analysis(symbol: str, date: str) -> dict:
    """运行完整分析链路"""
    print(f"开始分析 {symbol} {date}...")

    # Step 1: 采集数据
    print("1. 采集数据...")
    klines = fetch_klines(symbol, date)
    print(f"   获取到 {len(klines)} 条 K 线数据")

    # Step 2: 异常检测
    print("2. 检测异常...")
    anomalies = detect_anomalies(klines)
    print(f"   发现 {len(anomalies)} 个异常点")

    # Step 3: 情感分析
    print("3. 分析情感...")
    sentiment = analyze_sentiment(symbol, date)
    print(f"   情感得分: {sentiment['score']:.2f}")

    # Step 4: 归因报告
    print("4. 生成归因报告...")
    report = generate_report(symbol, date, anomalies, sentiment)
    print(f"   报告生成完成")

    return {
        "symbol": symbol,
        "date": date,
        "klines_count": len(klines),
        "anomalies": anomalies,
        "sentiment": sentiment,
        "report": report,
    }


if __name__ == "__main__":
    symbol = sys.argv[1] if len(sys.argv) > 1 else "600519"
    date = sys.argv[2] if len(sys.argv) > 2 else "2024-06-01"
    result = run_analysis(symbol, date)
    print("\n最终报告:")
    print(result["report"])
```

**运行方式：**

```bash
python backend/scripts/run_analysis.py 600519 2024-06-01
```

只要这个能跑通，整个项目的主干就打通了。

## 建议的开发顺序（最小可行路径）

每一步都是**可独立验证的**——不需要所有模块都就绪才能看到效果。

| 阶段 | 任务 | 验证方式 |
|------|------|---------|
| **Week 1** | `data/akshare_client.py` → 能拉数据入库 | `python scripts/collect_data.py` 能跑通 |
| **Week 2** | `db/models.py` + `api/v1/` → 能查数据 | `curl http://localhost:8000/api/v1/klines?code=600519` 有返回 |
| **Week 3** | `analysis/anomaly_detector.py` → 检出异常 | `detect_anomalies()` 有输出 |
| **Week 4** | `agent/graph.py` → LangGraph 串联链路 | `run_analysis.py` 能完整跑通 |
| **Week 5** | `analysis/sentiment.py` → 新闻情感接入 | 情感得分有输出 |
| **Week 6** | 整合优化 + 定时调度 + 通知 | 定时任务正常运行 |

**每周验收标准**：每周末都能独立验证这周新增的模块——能跑通、有输出、不报错。
