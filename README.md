## 一、项目概述

### 1.1 项目背景

在金融和加密资产领域，每天产生海量的时序数据（价格、利率、交易量等）。分析师面临的挑战不是"数据太少"，而是"数据太多，难以快速定位问题根源"。

**核心痛点**：

- 数据波动时，分析师需要手动排查多个维度才能找到原因
- 经验依赖严重，新人难以快速上手
- 缺乏系统化的归因分析工具

### 1.2 项目目标

构建一个**智能金融数据归因分析平台**，让用户能够：

1. **一句话提问**：输入"为什么比特币今天跌了？"或"利率为什么上涨？"
2. **自动检测**：系统自动检测数据异常，提取变化特征
3. **智能归因**：自动拆解各维度贡献度，定位核心驱动因素
4. **经验累积**：系统越用越聪明，自动匹配历史相似案例

### 1.3 核心价值

| 传统方式                   | 本平台                          |
| :------------------------- | :------------------------------ |
| 分析师手动排查，耗时数小时 | 秒级自动检测 + 归因分析         |
| 依赖个人经验               | 经验数字化、可复用              |
| 每次从头分析               | 历史经验自动匹配                |
| 报表静态、难以解读         | AI生成业务语言报告 + 可视化看板 |

## 三、核心功能模块

### 3.1 数据接入模块

**功能**：支持多种数据源接入，统一转换为内部数据模型

| 数据源类型 | 实现方式              | 金融场景示例              |
| :--------- | :-------------------- | :------------------------ |
| CSV文件    | `CsvDataAdapter`      | CalcFi导出的利率/汇率数据 |
| 数据库     | `DatabaseDataAdapter` | 已存在的金融数据库        |
| API        | `ApiDataAdapter`      | CalcFi Live API / 币安API |

**输入**：CSV/数据库表/API响应
**输出**：统一的`Dataset`对象（当前期 + 对比期）

### 3.2 异常检测模块（双引擎 + 经验增强）

**功能**：从多个维度检测数据异常，结合AI判断真假

**算法引擎（8种检测方法）**：

| 检测方法      | 适用场景 | 金融示例                |
| :------------ | :------- | :---------------------- |
| Z-score（3σ） | 偏离均值 | 利率单日暴涨超3倍标准差 |
| IQR（箱线图） | 离群值   | 交易量异常放大          |
| 同比偏差      | 年度对比 | 今年Q2 vs 去年Q2        |
| 环比偏差      | 月度对比 | 本月 vs 上月            |
| 方向检测      | 涨跌判断 | 比特币由涨转跌          |
| 速度检测      | 变化速率 | 跌幅加速扩大            |
| 拐点检测      | 趋势反转 | V型/倒V型反转           |
| 季节性检测    | 周期规律 | 年底资金面紧张          |

**AI决策层**：

- 输入：算法检测结果 + 信号特征 + 业务上下文
- 输出：真假异常判断 + 判断理由

**经验增强**：

- 检索历史相似模式，高匹配度直接复用结论
- 无匹配时走完整流程，并保存为新经验

### 3.3 归因分析模块（双引擎 + 经验增强）

**功能**：量化各维度对指标变化的贡献度

**算法引擎**：

- **基期拆解法**：拆解"价格 vs 数量"的贡献（适用于金融产品）
- **多维贡献度计算**：按区域/品类/时间/产品类型动态聚合
- **核心驱动因素排序**：识别最大贡献者和最大拖累者

**AI决策层**：

- 输入：算法计算结果 + 行业上下文
- 输出：业务洞察 + 可行动建议

**经验增强**：

- 检索历史归因模式，高匹配度直接复用归因结论
- 无匹配时走完整流程，并保存为新经验

### 3.4 经验累积模块

**功能**：让系统越用越聪明

**核心机制**：

| 机制           | 实现方式                         | 金融场景示例                     |
| :------------- | :------------------------------- | :------------------------------- |
| **经验向量化** | 20维特征向量（统计+信号+上下文） | "3月利率下跌"的模式被编码        |
| **相似检索**   | 余弦相似度匹配                   | 识别"今年3月"类似"去年3月"       |
| **置信度进化** | 用户反馈驱动                     | 正确判断+0.1，错误判断-0.15      |
| **自动学习**   | 每日聚类分析，提炼规则           | 发现"每年年底国债收益率"的规律   |

## 六、核心算法说明

### 6.1 异常检测算法

**Z-score（3σ准则）**：

text

```
Z = |X - μ| / σ
Z > 3 → 严重异常
Z > 2 → 轻微异常
```



**IQR（箱线图）**：

text

```
Q1 = 25%分位数, Q3 = 75%分位数
IQR = Q3 - Q1
下限 = Q1 - 1.5*IQR, 上限 = Q3 + 1.5*IQR
超出范围 → 异常
```



**同比/环比**：

text

```
同比 = (当前 - 去年同期) / 去年同期
环比 = (当前 - 上期) / 上期
变化率 > 阈值 → 异常
```



**拐点检测**：

text

```
连续N期方向变化 → 趋势反转
出现V型/倒V型 → 拐点
```



### 6.2 归因分析算法

**基期拆解法（价格×数量）**：

text

```
变化 = 当前 - 对比期
价格贡献 = (当前价格 - 对比期价格) × 对比期数量
数量贡献 = (当前数量 - 对比期数量) × 当前价格
总变化 = 价格贡献 + 数量贡献
```



**多维贡献度计算**：

text

```
维度值贡献度 = (当前值 - 对比期值) / 总变化 × 100%
```



### 6.3 经验向量化（20维）

## 十、总结

本项目构建了一个**面向金融数据的智能归因分析平台**，核心特色是：

1. **算法 + AI 双引擎**：算法保证精确计算，AI理解业务上下文
2. **经验累积机制**：系统越用越聪明，自动匹配历史经验
3. **多智能体协作**：LangGraph 编排异常检测 → 归因分析 → 报告生成
4. **垂直金融场景**：面向利率、汇率、加密货币等专业数据

**预期价值**：

- 将分析师数小时的人工排查缩短到秒级
- 将个人经验转化为系统可复用的数字化知识
- 用可视化看板 + AI解读降低数据解读门槛
